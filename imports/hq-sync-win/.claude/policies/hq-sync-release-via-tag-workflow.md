## Rule

The only sanctioned path to producing and publishing an hq-sync-win release is:

1. Bump the version triple (see `hq-sync-version-triple-lockstep`).
2. Commit + push the bump to `main`.
3. `git tag vX.Y.Z && git push origin vX.Y.Z`.
4. `.github/workflows/release.yml` runs on `windows-latest`, builds the per-arch matrix (`x86_64-pc-windows-msvc` + `aarch64-pc-windows-msvc`), Authenticode-signs every bundle with SignTool, generates `latest.json` with the Ed25519 updater signature, and creates a **draft** GitHub Release.
5. After draft-asset validation passes, the release is published via `gh release edit --draft=false` (also handled inside the workflow).

Local-machine `tauri build` outputs are for development and manual testing only — they MUST NOT be uploaded to GitHub Releases (`gh release upload`, drag-and-drop in the UI, or otherwise). The Tauri auto-updater relies on the `latest.json` file generated by `release.yml`; assets uploaded by hand will not match the signature manifest and the auto-updater will refuse them silently.

## Why

- `release.yml` is the only environment that holds the `WINDOWS_SIGNING_CERT` (.pfx) — a locally-built unsigned MSI/NSIS triggers SmartScreen "Unrecognized app" for every end user.
- The workflow generates `latest.json` with the right Ed25519 signature (`TAURI_SIGNING_PRIVATE_KEY`); locally-built bundles will fail signature verification and brick the auto-updater for the cohort.
- Sentry release + debug-symbol upload is wired into the workflow; manual builds skip it, and crash reports from that release become unreadable.
- The per-arch matrix (`x64` + `arm64`) is matrixed in CI; a local build emits only one triple, which would leave the other arch on a stale auto-updater feed.

## How to comply

- Treat `git tag vX.Y.Z && git push origin vX.Y.Z` as the only release action.
- If the workflow fails, fix the failure and retry the tag — never paper over by uploading a local build.
- Manual `workflow_dispatch` with a tag input is allowed (the workflow recomputes everything from the tag).

## Exceptions

None. A future release channel (e.g. nightly) must add its own workflow rather than bypassing this one.
